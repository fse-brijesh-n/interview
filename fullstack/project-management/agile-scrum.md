# Agile & Scrum Interview Q&A

## Overview
Agile and Scrum cover iterative development, sprint planning, user stories, retrospectives, and collaborative practices for delivering value incrementally with flexibility.

---

## Interview Questions & Answers

1. **What is Agile?**
   - Iterative development methodology prioritizing flexibility, customer collaboration, and continuous improvement.

2. **What are Agile Principles?**
   - From Agile Manifesto: individuals over processes, working software over documentation, customer collaboration over contracts, responding to change.

3. **What is Scrum?**
   - Framework implementing Agile with defined roles, ceremonies, and artifacts.

4. **What are Scrum Roles?**
   - Product Owner, Scrum Master, Development Team.

5. **What is Product Owner?**
   - Responsible for product vision, requirements, and prioritization.

6. **What is Scrum Master?**
   - Facilitator ensuring team follows Scrum practices.

7. **What is Development Team?**
   - Cross-functional group delivering increments.

8. **What is User Story?**
   - Feature from user perspective: "As <role>, I want <feature> so that <benefit>".

9. **What is Acceptance Criteria?**
   - Conditions defining when user story is complete.

10. **What is Sprint?**
    - Fixed time-box (1-4 weeks) for development work.

11. **What is Sprint Planning?**
    - Meeting defining sprint goals and selecting user stories.

12. **What is Story Point?**
    - Relative size estimate for user story.

13. **What is Velocity?**
    - Amount of work completed per sprint.

14. **What is Backlog?**
    - Prioritized list of features and improvements.

15. **What is Product Backlog?**
    - Complete list of features for product.

16. **What is Sprint Backlog?**
    - User stories selected for current sprint.

17. **What is Daily Standup?**
    - 15-minute daily meeting discussing progress and blockers.

18. **What is Sprint Review?**
    - Demonstrating completed work to stakeholders.

19. **What is Sprint Retrospective?**
    - Reflecting on sprint improvements.

20. **What is Definition of Done?**
    - Criteria defining when work is complete.

21. **What is Burndown Chart?**
    - Showing work remaining vs time in sprint.

22. **What is Burnup Chart?**
    - Showing work completed vs time.

23. **What is Refinement?**
    - Clarifying and estimating backlog items.

24. **What is Epic?**
    - Large feature spanning multiple sprints.

25. **What is Theme?**
    - Collection of related epics and stories.

26. **What is Spike?**
    - Investigation story exploring technical solution.

27. **What is Technical Debt?**
    - Accumulated shortcuts requiring future work.

28. **What is Impediment?**
    - Blocker preventing progress.

29. **What is Velocity Trend?**
    - Tracking velocity changes over sprints.

30. **What is Sprint Goal?**
    - Objective team commits to for sprint.

31. **What is Done Increment?**
    - Potentially shippable product at sprint end.

32. **What is Continuous Integration?**
    - Frequent code integration with automated tests.

33. **What is Continuous Delivery?**
    - Always maintaining deployable state.

34. **What is Continuous Deployment?**
    - Automatically deploying to production.

35. **What is Test-Driven Development?**
    - Writing tests before implementation.

36. **What is Pair Programming?**
    - Two developers writing code together.

37. **What is Code Review?**
    - Peer examination before merging.

38. **What is Refactoring?**
    - Improving code without changing functionality.

39. **What is Kanban?**
    - Visual management system for workflow.

40. **What is Kanban Board?**
    - Board showing workflow stages: To Do, In Progress, Done.

41. **What is WIP Limit?**
    - Maximum work in progress at each stage.

42. **What is Lean?**
    - Eliminating waste and optimizing flow.

43. **What is Flow?**
    - Smooth progression of work through system.

44. **What is Lead Time?**
    - Time from idea to production.

45. **What is Cycle Time?**
    - Time for work item from start to done.

46. **What is Throughput?**
    - Number of items completed per period.

47. **What is Slack?**
    - Buffer capacity for unexpected work.

48. **What is Escalation?**
    - Raising issues to management.

49. **What is Facilitation?**
    - Guiding group discussions effectively.

50. **What is Empowerment?**
    - Giving team authority for decisions.

51. **What is Cross-Functional Team?**
    - Team with all skills needed for work.

52. **What is Self-Organizing Team?**
    - Team deciding how to accomplish goals.

53. **What is Team Velocity?**
    - Consistent output over multiple sprints.

54. **What is Backlog Grooming?**
    - Maintaining and refining backlog.

55. **What is Release Planning?**
    - Planning software release timeline.

56. **What is Roadmap?**
    - High-level product direction over time.

57. **What is Stakeholder?**
    - Person with interest in project success.

58. **What is Business Value?**
    - Impact on organization's goals.

59. **What is Return on Investment (ROI)?**
    - Measuring value relative to cost.

60. **What is Iterative Development?**
    - Repeating cycles of planning, development, review.

## Scrum Ceremonies

| Ceremony | Duration | Frequency | Purpose |
|----------|----------|-----------|---------|
| **Sprint Planning** | 4 hours | Per sprint | Planning sprint work |
| **Daily Standup** | 15 min | Daily | Synchronization |
| **Sprint Review** | 2 hours | Per sprint | Demonstrating done work |
| **Retrospective** | 1.5 hours | Per sprint | Improvement discussion |

---

## Best Practices

1. Keep sprints consistent length
2. Prioritize ruthlessly
3. User stories should be small
4. Maintain definition of done
5. Respect sprint commitments
6. Embrace continuous improvement
7. Use velocity for planning
8. Visualize work
9. Communicate transparently
10. Inspect and adapt

---

## Common Mistakes

1. Too large sprints
2. Incomplete user stories
3. Ignoring velocity trends
4. Gold-plating features
5. Not releasing regularly
6. Poor prioritization
7. Scope creep mid-sprint
8. No definition of done
9. Skipping retrospectives
10. Ignoring technical debt

| **Fibonacci** | Relative sizing | Medium |
| **Three-Point** | Risk accounting | High |

---

## Summary

Understand Scrum roles, ceremonies, and artifacts. Master user story writing and estimation. Follow sprint rhythm: planning → daily standups → review → retrospective. Use velocity for predictability. Maintain backlog, implement CI/CD, and focus on delivering value incrementally.
